# Библиотека компонентов визуализации данных

Системное руководство по выбору и реализации эффективных визуализаций: сопоставление характера данных с подходящим типом визуализации при сохранении ясности, доступности и силы воздействия.

## Обзор

Визуализация данных превращает сырые данные в визуальные представления, которые вскрывают паттерны, тренды и инсайты. Этот скилл дает:

1. **Фреймворк выбора**: системные деревья решений «тип данных + цель → тип графика»
2. **24+ метода визуализации**: сгруппированы по аналитической цели
3. **Паттерны доступности**: соответствие WCAG 2.1 AA, палитры для дальтоников
4. **Стратегии производительности**: оптимизация под объем данных (<1000 до >100K точек)
5. **Поддержку вывода в документ**: структурированные шаблоны статистических отчетов `.docx`
6. **Режим критики**: рубрика из 7 измерений для ревью и починки готовых графиков (Шаг 7)
7. **HTML-отчеты для шаринга**: автономные одностраничные отчеты с inline-SVG графиками
8. **Многоязычность**: JavaScript/TypeScript (основной), Python, Rust, Go

## Принцип «сначала сообщение» (Zelazny)

До любой механики графика прогони пайплайн McKinsey/Zelazny:

```
ДАННЫЕ → 1. СООБЩЕНИЕ → 2. СРАВНЕНИЕ → 3. ТИП ГРАФИКА
```

Тип графика определяется твоим **сообщением** (единственной мыслью на вынос), а не самими данными. Два правила без компромиссов:

1. **Сначала сформулируй сообщение одним утвердительным предложением.** Не можешь — значит, мысли еще нет, график не выбираешь.
2. **Заголовок констатирует вывод, а не тему.** Заголовок-действие (`"Объем продаж удвоился"`) вместо ярлыка-темы (`"Динамика продаж"`). Один график = одно сообщение.

**Детектор типа сравнения** (сообщение → сравнение → график по умолчанию):

| Сравнение | Слова-триггеры | График по умолчанию |
|---|---|---|
| **Компонент** (% от целого) | доля, % от целого, составило X% | Pie (≤6 секторов) / 100% stacked bar |
| **Позиция/элемент** (ранжирование) | больше/меньше, на N-м месте, largest | Bar (отсортированный) |
| **Временной ряд** (изменение) | рос/снижался/колебался, trend | Column (≤6 точек) / Line (много) |
| **Частота** (счет по диапазонам) | в диапазоне X–Y, распределение | Histogram (мало корзин) / кривая |
| **Корреляция** (связь) | зависит/не зависит, связь между | Парные bars (мало) / Scatter (много) |

**Презентация vs документ:** график на слайде должен быть **в 2× проще и в 4× яснее**, чем тот же график в письменном отчете. Для **качественных/нечисловых** идей (процесс, барьеры, баланс) используй **визуальные концепты/метафоры**, а не графики данных.

**Полный фреймворк:** см. `references/zelazny-message-framework.md`

## Быстрый старт: рабочий процесс

### Шаг 0: Определи сообщение
```
Сформулируй мысль на вынос ОДНИМ предложением → оно станет заголовком-действием графика.
Определи тип сравнения (таблица выше) → график по умолчанию.
```

### Шаг 1: Оцени данные
```
Какой тип? [категориальные | непрерывные | временные | пространственные | иерархические]
Сколько измерений? [1D | 2D | многомерные]
Сколько точек? [<100 | 100-1K | 1K-10K | >10K]
```

### Шаг 2: Определи цель
```
Какую историю рассказать? [сравнение | тренд | распределение | связь | состав | поток | иерархия | география]
```

### Шаг 3: Выбери тип графика

**Быстрый выбор:**
- Сравнить 5-10 категорий → Bar Chart
- Показать продажи за 12 месяцев → Line Chart
- Показать распределение возрастов → Histogram или Violin Plot
- Исследовать корреляцию → Scatter Plot
- Показать разбивку бюджета → Treemap или Stacked Bar
- Нужен управленческий deliverable в формате Word → шаблон статистического отчета `.docx`
- Нужно покрытие сменами по часам/дням с интерактивом → Staff Coverage Heatmap (HTML)

**Полные деревья решений:** см. `references/selection-matrix.md`

### Шаг 4: Реализуй

См. разделы по языкам ниже с рекомендованными библиотеками.

### Шаг 5: Примени доступность
- Добавь текстовую альтернативу (aria-label)
- Обеспечь контраст цвета минимум 3:1
- Используй палитру, безопасную для дальтоников
- Дай альтернативу в виде таблицы данных

### Шаг 6: Оптимизируй производительность
- <1000 точек: стандартный рендеринг SVG
- >1000 точек: сэмплирование или рендеринг через Canvas
- Очень много: серверная агрегация

### Шаг 7: Критика (ревью готового или только что построенного графика)
Не выпускай без прохода-ревью. Оцени график по рубрике из 7 измерений — сообщение, соответствие типа графика, кодирование/восприятие, подписи, цвет/доступность, data-ink, сторителлинг — найди что сломано и **пересобери с правками**. График для презентации должен набирать ≥12/14 без нулей.

**Полная рубрика и формат вывода:** см. `references/critique-rubric.md`

### Режим вывода: одностраничный HTML-отчет для шаринга
Для портативного deliverable без зависимостей (живой дашборд, прототип-сравнение, выжимка для руководства), который открывается офлайн и хостится как статический файл, собери **автономный `.html`** с inline-SVG графиками.

**Шаблон, скелет и заметки по деплою:** см. `references/html-report-template.md`

## Каталог визуализаций

### Уровень 1: Базовые примитивы
Широкая аудитория, прямолинейные истории на данных:
- **Bar Chart**: сравнение категорий
- **Line Chart**: тренды во времени
- **Scatter Plot**: исследование связей
- **Pie Chart**: часть-к-целому (макс. 5-6 секторов)
- **Area Chart**: акцент на величине во времени

### Уровень 2: По назначению
Конкретные аналитические инсайты:
- **Сравнение**: Grouped Bar, Lollipop, Bullet Chart
- **Тренд**: Stream Graph, Slope Graph, Sparklines
- **Распределение**: Violin Plot, Box Plot, Histogram
- **Связь**: Bubble Chart, Hexbin Plot
- **Состав**: Treemap, Sunburst, Waterfall
- **Поток**: Sankey Diagram, Chord Diagram

### Уровень 3: Продвинутые
Сложные данные, искушенная аудитория:
- **Многомерные**: Parallel Coordinates, Radar Chart, Small Multiples
- **Временные**: Gantt Chart, Calendar Heatmap, Candlestick
- **Сетевые**: Force-Directed Graph, Adjacency Matrix

**Детальные описания:** см. `references/chart-catalog.md`

## Требования доступности (WCAG 2.1 AA)

### Текстовые альтернативы
```html
<figure role="img" aria-label="Sales increased 15% from Q3 to Q4">
  <svg>...</svg>
</figure>
```

### Требования к цвету
- Нетекстовые UI-элементы: контраст минимум 3:1
- Текст: минимум 4.5:1 (или 3:1 для крупного текста ≥24px)
- Не полагайся только на цвет — добавляй паттерны/текстуры + подписи

### Палитры, безопасные для дальтоников

**Палитра IBM (рекомендуется):**
```
#648FFF (синий), #785EF0 (фиолетовый), #DC267F (маджента),
#FE6100 (оранжевый), #FFB000 (желтый)
```

**Избегай:** сочетаний красный/зеленый (8% мужчин имеют красно-зеленый дальтонизм)

### Навигация с клавиатуры
- Tab для перехода по интерактивным элементам
- Enter/Space для активации подсказок
- Стрелки для навигации по точкам данных

**Полный гайд по доступности:** см. `references/accessibility.md`

## Реализация на JavaScript/TypeScript

### Recharts (бизнес-дашборды)
Композируемые React-компоненты, декларативный API, адаптивность по умолчанию.

```bash
npm install recharts
```

```tsx
import { LineChart, Line, XAxis, YAxis, Tooltip, ResponsiveContainer } from 'recharts';

const data = [
  { month: 'Jan', sales: 4000 },
  { month: 'Feb', sales: 3000 },
  { month: 'Mar', sales: 5000 },
];

export function SalesChart() {
  return (
    <ResponsiveContainer width="100%" height={300}>
      <LineChart data={data}>
        <XAxis dataKey="month" />
        <YAxis />
        <Tooltip />
        <Line type="monotone" dataKey="sales" stroke="#8884d8" />
      </LineChart>
    </ResponsiveContainer>
  );
}
```

### D3.js (кастомные визуализации)
Максимальная гибкость, индустриальный стандарт, неограниченные типы графиков.

```bash
npm install d3
```

### Plotly (научные/интерактивные)
3D-визуализации, статистические графики, интерактив «из коробки».

```bash
npm install react-plotly.js plotly.js
```

### Docx (статистические отчеты Word)
Используй для бизнес-готовой отчетности `.docx` с таблицами метрик, цветовой раскраской дельт и сигнальными индикаторами (`🔴`, `✅`, `Норма`).

```bash
npm install docx
```

Ключевые стилевые ограничения этого шаблона:
- Times New Roman по всему документу
- H1/H2 цвет `#2E5496`
- Шапка таблицы `#2E5496` с белым текстом
- Чередование строк таблицы `#EEF3FB` / белый
- Цвета дельт: красный `#C0392B`, зеленый `#27AE60`, нейтральный `#E67E22`

Детальный гайд: `references/statistical-reports-docx.md`

### Staff Coverage Heatmap (Python + HTML)
Используй для операционного анализа укомплектованности из таблиц смен (Excel/CSV): тепловая карта час/день, детали при наведении, метрики покрытия.

Базовый пайплайн:
1. Распарсить смены (`08:00-20:00`, `в`, `ОТПУСК`) в диапазоны времени
2. Построить матрицу покрытия `hours x days`
3. Посчитать сводные метрики (макс., средн., % покрытия, пиковый час)
4. Отрендерить интерактивную HTML тепловую карту

Детальный гайд: `references/staff-coverage-heatmap.md`

## Частые ошибки, которых стоит избегать

0. **Нет сообщения** — выбор графика до формулировки мысли одним предложением (шаг 1 Zelazny)
0b. **Заголовок-тема вместо заголовка-сообщения** — "Продажи по регионам" прячет суть; используй "Регион B дает 40% продаж"
1. **Мышление «сначала график»** — выбирай исходя из данных + цели, а не эстетики
2. **Pie charts для >6 категорий** — используй отсортированный bar chart
3. **Графики с двумя осями** — обычно вводят в заблуждение, используй small multiples
4. **3D, когда хватает 2D** — добавляет сложности, снижает ясность
5. **Радужные цветовые шкалы** — не равномерны перцептивно, не безопасны для дальтоников
6. **Обрезанная ось Y** — обозначь явно или начинай с нуля
7. **Слишком много цветов** — ограничься 6-8 различимыми категориями
8. **Нет контекста** — всегда подписывай оси, указывай единицы

## Быстрое дерево решений

```
СТАРТ: что у тебя за данные?

Категориальные (категории/группы)
  ├─ Сравнить значения → Bar Chart
  ├─ Показать состав → Treemap или Pie Chart (<6 секторов)
  └─ Показать поток → Sankey Diagram

Непрерывные (числа)
  ├─ Одна переменная → Histogram, Violin Plot
  └─ Две переменные → Scatter Plot

Временные (временные ряды)
  ├─ Одна метрика → Line Chart
  ├─ Несколько метрик → Small Multiples
  └─ Дневные паттерны → Calendar Heatmap

Иерархические (вложенные)
  ├─ Пропорции → Treemap
  └─ Показать глубину → Sunburst, Dendrogram

Географические (локации)
  ├─ Региональные агрегаты → Choropleth Map
  └─ Точечные локации → Symbol Map
```
